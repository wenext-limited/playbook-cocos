# Plan: cocos-feature-migration 调度可靠性改造

## 目标

按用户要求，把 skill 的 agent 调度升级为：

`checkpoint + state bootstrap + heartbeat + watchdog + artifact harvest + DAG transition`

核心目标是：即使子 agent 不回 `agent_result`、只发 idle、或长时间运行，主控也能通过文件事实和 watchdog 自动收割/推进，不再依赖用户询问。

## 关键设计细节

### A. State bootstrap

主控启动任何阶段 agent 前，必须先写最小 running state compact：

```yaml
status: running
execution_status: running
phase_runtime:
  phase: <phase>
  agent_name: <agent>
  agent_id: pending_until_spawned
  lease_status: active
  restart_count: 0
  required_artifacts_pending: [...]
  current_step: controller_spawn_prepared
```

这样 agent 运行期间，主控/用户都能看到当前阶段不是 missing，而是 running。

### B. Heartbeat

每个 agent 使用单独 heartbeat 文件：

`<source_analysis_dir-or-target_migration_dir>/logs/heartbeat/<phase>-<agent>.heartbeat.json`

只允许写调度信息，不允许写完整资源/代码清单：

```json
{
  "phase": "04-source-resource-closure",
  "agent": "source-resource-closure-analyzer",
  "status": "running",
  "current_step": "asset deps/uuid/refs refresh",
  "completed_steps": 3,
  "total_known_steps": 5,
  "last_output_path": "logs/asset-deps-PanelGeneralRank-prefab.txt",
  "updated_at": "..."
}
```

### C. Watchdog

主控启动 agent 后必须启动 bounded watchdog：

- 优先 `Monitor`。
- 若 `Monitor` 不可用，使用 `Bash(run_in_background=true)` 的一-shot fallback。
- watchdog 只在 checkpoint、complete、soft_timeout、failure 四类状态输出单行事件。
- 禁止输出完整日志、完整 compact、完整资源清单。

示例单行事件：

```text
phase=04-source-resource-closure event=completed artifacts_ok=true state=completed
```

### D. Artifact harvest

主控收到 agent_result / idle / watchdog 事件 / 用户询问 / resume 任一触发后，统一执行：

1. 读 checkpoint。
2. 检查 required artifacts 是否存在。
3. 读当前 phase state compact。
4. 若 artifacts + state 完整：记录 `completed_with_agent_output_missing`（如无 agent_result），并进入 DAG transition。
5. 若缺失且未追问：追问一次。
6. 追问后仍缺失：同阶段最多重启一次。
7. 重启后仍缺失：阻塞并写 manifest 风险。

### E. DAG transition

显式记录每个阶段完成后的下一跳。

特别是第 5 步：

```text
04 completed -> generate/refresh 05x shared search -> start 05a
05a completed -> start 05b + 05c
05b + 05c completed -> controller merge 05a/05b/05c
05 merge completed + no open confirmation -> start 06
06 completed -> start 07 static-verifier
07 verifier completed -> start final-report-writer
```

### F. 上下文预算

新增硬限制：

- checkpoint 默认 <= 80 行。
- heartbeat 默认 <= 40 行。
- watchdog 每条事件 <= 1 行。
- state compact 默认 <= 200 行。
- event log 默认不读；冲突/恢复时最多读最近 80 行。
- artifact harvest 只做存在性检查 + 读 state compact。
- 禁止 heartbeat/watchdog 输出完整证据。

## 风险与处理

- `Monitor` 可能不是所有环境都允许：规则中保留 Bash background fallback。
- watchdog 事件过多可能污染上下文：限制只在 checkpoint/complete/timeout/failure 输出。
- agent prompts 数量较多：采用统一块批量插入，保持各 prompt 原有职责不变。
- 现有迁移产物不兼容：Resume 规则允许旧产物缺 heartbeat，标记 `legacy_no_heartbeat` 后继续。
