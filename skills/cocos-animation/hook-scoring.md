# cocos-animation 技能评分标准

## 目的

本评分标准用于对 `cocos-animation` 技能进行持续优化，评估它在指导 Cocos Creator 2D 动画开发时是否足够准确、完整、可执行，并能否约束常见动画风险。

评分时不要只看文档篇幅，要看技能是否能让代理在真实任务中产出更安全、更符合项目习惯的动画代码。

## 使用方式

当需要优化或审查 `cocos-animation/SKILL.md` 时，按本文档逐项评分：

1. 先阅读当前 `SKILL.md`。
2. 按 8 个维度给出 0-5 分。
3. 计算总分：各维度得分 × 权重。
4. 针对低于 4 分的维度提出具体补强项。
5. 修改 `SKILL.md` 后重新评分，确认改动确实提升了可用性。

## 总分等级

| 总分 | 等级 | 含义 | 动作 |
|---|---|---|---|
| 90-100 | A | 可直接作为高质量技能使用 | 仅做小幅维护 |
| 80-89 | B | 基本可靠，少量场景需补强 | 优先补齐低分维度 |
| 70-79 | C | 能用但指导不稳定 | 需要结构化优化 |
| 60-69 | D | 覆盖不足或风险约束薄弱 | 需要重写关键章节 |
| < 60 | F | 不适合作为技能触发依据 | 重新设计技能 |

## 评分维度

### 1. 适用场景与边界清晰度（10%）

评估技能是否说明什么时候该用、什么时候不该用。

| 分数 | 标准 |
|---|---|
| 5 | 明确覆盖 Tween、Spine、timer、update、预回调等场景，并清楚排除粒子/3D/资源制作等非目标范围 |
| 4 | 场景较清晰，但少量边界需要推断 |
| 3 | 只列出主要动画类型，缺少选择依据或反例 |
| 2 | 场景描述泛泛，容易误用于不相关任务 |
| 1 | 只有主题名，没有可操作边界 |
| 0 | 与动画技能目标不匹配 |

优化建议：补充“适用/不适用”小节和动画类型选择表。

### 2. 动画类型选择决策质量（15%）

评估技能能否帮助代理选择合适实现方式。

| 分数 | 标准 |
|---|---|
| 5 | 对 Tween、Spine、timer、update、贝塞尔、预回调给出明确选择规则、原因和典型需求 |
| 4 | 有选择表，但对复杂组合场景说明不足 |
| 3 | 能区分基本类型，但无法指导复杂流程 |
| 2 | 只有 API 示例，没有决策依据 |
| 1 | 容易让代理把所有动画都写成同一种方式 |
| 0 | 没有动画类型选择信息 |

优化建议：增加“需求 → 推荐方式 → 不推荐方式 → 原因”的对照。

### 3. 生命周期与资源安全（15%）

评估技能是否能防止动画相关泄漏、销毁后回调、重复触发等问题。

| 分数 | 标准 |
|---|---|
| 5 | 明确要求清理 Tween、原生定时器、Spine 回调、预回调引用，并在异步回调中检查节点/组件有效性 |
| 4 | 覆盖主要清理点，但缺少某类回调或有效性检查 |
| 3 | 只有清单提示，示例代码没有体现 |
| 2 | 只提到部分清理，如只提 `Tween.stopAllByTarget` |
| 1 | 生命周期约束非常弱 |
| 0 | 没有生命周期安全内容 |

优化建议：为每类动画增加 `onDestroy` 清理模板和回调有效性检查示例。

### 4. 异步流程协调能力（15%）

评估技能是否能指导动画完成后接业务逻辑，避免时序错误。

| 分数 | 标准 |
|---|---|
| 5 | 系统解释直接回调、预回调、链式预回调、Spine complete、Tween call 的取舍，并强调完成点唯一性 |
| 4 | 预回调模式清楚，但缺少与 Tween/定时器混合时序的说明 |
| 3 | 有完成回调示例，但缺少跨动画流程编排 |
| 2 | 只展示 callback 写法，没有时序原则 |
| 1 | 容易导致动画未完成就执行后续逻辑 |
| 0 | 没有动画完成处理内容 |

优化建议：补充“后续业务必须挂在真实完成点”的规则，以及多段动画串联模板。

### 5. 项目规范符合度（15%）

评估技能是否遵守当前项目的 Cocos/TypeScript 编码习惯。

| 分数 | 标准 |
|---|---|
| 5 | 示例遵守项目命名、宽松等号要求、魔法数字提取、中文 WHY 注释、节点/组件有效性检查和常量化约定 |
| 4 | 大体符合，但少量示例仍有裸数字或注释偏 WHAT |
| 3 | 规范只在文字中提到，示例未完全体现 |
| 2 | 示例含较多硬编码时长、偏移、状态值 |
| 1 | 与项目习惯明显冲突 |
| 0 | 完全没有项目规范约束 |

优化建议：把动画时长、延迟、速度表、阈值等示例数字抽成配置常量。

### 6. 示例代码可迁移性（10%）

评估示例是否能直接迁移到真实 Cocos 组件中。

| 分数 | 标准 |
|---|---|
| 5 | 示例包含必要 import、字段声明、初始化、播放、清理和边界检查，复制后只需替换业务名 |
| 4 | 示例主体可用，但缺少少量上下文 |
| 3 | 示例能表达思路，但需要较多补全 |
| 2 | 示例片段过散，难以落地 |
| 1 | 示例容易误导实现 |
| 0 | 没有可用示例 |

优化建议：为核心模式提供完整组件片段，而不是只有局部调用。

### 7. 常见风险覆盖度（10%）

评估技能是否覆盖动画开发中的高频坑。

| 分数 | 标准 |
|---|---|
| 5 | 覆盖重复播放、并行动画互相覆盖、节点销毁、暂停恢复、循环动画停止、状态重置、速度表越界、Spine 动画名判断等风险 |
| 4 | 覆盖大多数风险，少量边界缺失 |
| 3 | 有清单但风险解释不充分 |
| 2 | 只覆盖一两个明显风险 |
| 1 | 风险提示过于笼统 |
| 0 | 没有风险提示 |

优化建议：增加“风险 → 症状 → 推荐处理”的表格。

### 8. 输出约束与检查清单（10%）

评估技能是否能约束代理最终交付物。

| 分数 | 标准 |
|---|---|
| 5 | 明确要求输出动画方案、关键代码、清理点、完成回调、常量位置和验证建议 |
| 4 | 有清单，但缺少交付格式 |
| 3 | 清单较短，只能做最终自查 |
| 2 | 输出要求不明确 |
| 1 | 无法判断任务是否完成 |
| 0 | 没有任何输出约束 |

优化建议：增加“回答/改代码前必须确认的信息”和“完成后必须说明的内容”。

## 扣分项

出现以下问题时，在对应维度外额外扣分，总分最低为 0：

| 问题 | 扣分 |
|---|---:|
| 示例中大量内联动画时长、延迟、坐标偏移等魔法数字 | -5 |
| 建议使用 `===` / `!==` 比较状态、uid、动画名等项目值 | -5 |
| 无限 Tween 或原生 timer 没有清理要求 | -10 |
| 动画完成点不明确，可能导致业务逻辑提前执行 | -10 |
| Spine/Tween 回调中没有任何销毁安全提示 | -5 |
| 声称支持当前技能未覆盖的粒子或 3D 动画实现 | -5 |
| 示例与 Cocos Creator 3.x API 明显不符 | -10 |

## 优化闭环

每次根据评分修改 `SKILL.md` 后，必须重新检查：

- 低分维度是否被具体内容修复，而不是只增加标题。
- 新增示例是否仍符合项目规范。
- 新增规则是否能直接影响代理行为。
- `SKILL.md` 是否保持动画技能聚焦，没有扩散到粒子、3D 或资源制作。
