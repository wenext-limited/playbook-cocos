# AGENTS.md

本仓库是面向 AI 代理的知识库，用于指导 Cocos Creator 2D 游戏项目的开发。包含技能（skills）、规则（rules）和架构文档，不含可运行的应用代码。

## 阅读顺序

1. 先读 `README.md` 了解整体概述。
2. 根据任务查看 `skills/<name>/SKILL.md`。
3. 仅在任务涉及共享约定时阅读 `rules/` 下的文档。
4. 查看 `agents/` 了解代理角色定义和技能路由。
5. 查看 `docs/` 了解项目架构全景。

## 结构说明

- `skills/` — 可复用的领域指导包。每个技能目录包含：
  - `SKILL.md` 作为主入口文件
  - 可选的 `references/` 存放参考资料
- `rules/` — 多个技能共享的仓库级约定。
- `agents/` — 仓库级代理资产（角色定义与技能路由）。
- `docs/` — 架构设计文档和实施说明。

## 编辑规则

- 不要编造项目中不存在的构建命令、运行时行为或应用设置。
- 保持编辑与当前文档优先的项目定位一致。
- 保持每个文档聚焦一个关注点的模式。
- 扩展技能时，同步更新技能的描述、工作流和引用，保持入口文件的连贯性。
- 大型结构变更前，先在 `docs/` 下添加设计说明。

## 技能命名规范

每个技能文件夹和 `SKILL.md` 的 frontmatter `name` 必须使用规定前缀。

### 允许的前缀

- `cocos-`：Cocos Creator 引擎通用，包括场景管理、UI 系统、动画、音频、资源加载、物理等。
- `oops-`：OOPS 框架（oops-plugin-framework）相关，包括 ECS 架构、事件系统、框架初始化等。
- `ts-`：TypeScript 语言级别的可复用指导，不绑定特定框架或引擎。

### 命名格式

- 目录：`skills/<prefix>-<topic>/`
- frontmatter：`name: <prefix>-<topic>`
- `topic` 使用简洁的 kebab-case。

### 前缀选择规则

1. 如果技能主要关于 OOPS 框架的用法、ECS 模式或框架特有功能，选 `oops-`。
2. 如果技能主要关于 Cocos Creator 引擎能力（场景、UI、动画、音频、资源等），选 `cocos-`。
3. 如果技能是语言级别的 TypeScript 指导，不依赖特定引擎或框架，选 `ts-`。

### 一致性要求

- 目录名、`SKILL.md` 的 `name` 字段保持一致。
- 跨技能引用时使用 `skills/<name>` 路径。
- 更新技能名称后同步更新所有交叉引用。

## 文件级要求

### Skills

- `skills/<name>/SKILL.md` 是规范入口。
- 参考资料应保持精准，直接关联技能主题。

### Rules

- 将广泛可复用的指导放在 `rules/` 中，避免在多个技能中重复。
- 修改共享规则时要谨慎，因为它可能影响多个技能。

### Docs

- 架构和设计文档放在 `docs/` 下。
- 保持描述足够清晰，便于后续贡献者理解意图。

## 文档变更验证清单

完成文档任务前的检查：

1. 重新阅读每个编辑过的文件。
2. 确认每个引用路径存在。
3. 确认摘要与当前仓库内容一致。
4. 检查配套文件是否同步更新。
5. 不要声称支持仓库中不存在的文件、工具或工作流。
